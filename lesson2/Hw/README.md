# Домашнее задание 2. ООП в PHP

## Описание задачи

Необходимо реализовать простую ООП-систему для генерации HTML-разметки на PHP.

Система должна позволять создавать HTML-элементы через классы, добавлять к ним атрибуты, вкладывать одни элементы в другие, выводить итоговый HTML-код и проверять ошибки валидации.

Главная идея задания: HTML-документ должен собираться не строками напрямую, а через объекты.

Нужно реализовать:

- одиночные теги;
- парные теги;
- текстовые узлы;
- специальные классы для отдельных тегов, например `img` и `a`;
- валидацию;
- вывод ошибок;
- защиту от вставки HTML и JavaScript.

## Пример ожидаемого результата

Пример HTML, который может быть сгенерирован системой:

```html
<div class="h1">
    <img>
    <hr>
    <h2>Hello, &lt;script&gt;alert(&quot;haha&quot;);&lt;/script&gt;, World!Поиск</h2>
    <a href="http://yandex.ru">Поиск</a>
</div>
```

Важно: строка со скриптом должна быть выведена как обычный текст, а не выполниться как JavaScript.

---

## Что нужно реализовать

## 1. Функция `report()`

Функция должна собирать ошибки валидации.

Требования:

- если передан текст ошибки, сохранить его;
- если функция вызвана без аргументов, вывести все накопленные ошибки;
- ошибки можно выводить через `print_r`.

Пример ошибок:

```txt
Attribute onclick is not allowed for SingleTag
TextNode is empty
Attribute href is required for A
```

Пример логики:

```php
function report(?string $message = null)
{
    static $errors = [];

    if ($message !== null) {
        $errors[] = $message;
    } else {
        print_r($errors);
    }
}
```

---

## 2. Абстрактный класс `Node`

`Node` — это базовый класс для всех элементов HTML-дерева.

От него должны наследоваться:

- `Tag`;
- `TextNode`.

Класс должен задавать общий интерфейс для рендера и валидации.

Пример:

```php
abstract class Node
{
    abstract public function render(): string;

    public function isValid(): bool
    {
        return true;
    }
}
```

---

## 3. Абстрактный класс `Tag`

`Tag` — базовый класс для HTML-тегов.

Класс должен хранить:

- имя тега;
- массив атрибутов;
- список разрешенных атрибутов;
- список обязательных атрибутов, если они нужны.

Должны быть реализованы методы:

- добавление атрибута;
- проверка разрешенности атрибута;
- проверка обязательных атрибутов;
- формирование строки атрибутов;
- валидация тега.

Метод добавления атрибута должен возвращать `$this`, чтобы можно было использовать цепочки вызовов.

Пример:

```php
$tag->attr('class', 'title')->attr('id', 'main-title');
```

---

## 4. Класс `SingleTag`

`SingleTag` — класс для одиночных HTML-тегов.

Примеры одиночных тегов:

```html
<img>
<hr>
<br>
```

Класс должен:

- наследоваться от `Tag`;
- выводить только открывающий тег;
- не иметь дочерних элементов.

Пример:

```php
$hr = new SingleTag('hr');
echo $hr->render();
```

Результат:

```html
<hr>
```

---

## 5. Класс `PairTag`

`PairTag` — класс для парных HTML-тегов.

Примеры парных тегов:

```html
<div></div>
<h2></h2>
<a></a>
```

Класс должен:

- наследоваться от `Tag`;
- хранить массив дочерних элементов;
- иметь метод `appendChild()`;
- выводить открывающий тег;
- выводить дочерние элементы;
- выводить закрывающий тег;
- проверять валидность всех дочерних элементов.

Пример:

```php
$div = new PairTag('div');
$h2 = new PairTag('h2');

$div->appendChild($h2);
```

Метод `appendChild()` должен возвращать `$this`.

---

## 6. Класс `TextNode`

`TextNode` — класс для текстового содержимого.

Требования:

- текст не должен быть пустым;
- текст должен экранироваться;
- HTML и JavaScript не должны выполняться.

Пример тестового текста:

```php
new TextNode('Hello, <script>alert("haha");</script>, World!');
```

На странице этот текст должен отобразиться безопасно, без выполнения JavaScript.

Для экранирования можно использовать:

```php
htmlspecialchars($text, ENT_QUOTES | ENT_SUBSTITUTE, 'UTF-8')
```

---

## 7. Класс `Img`

`Img` — отдельный класс для тега изображения.

Требования:

- наследуется от `SingleTag`;
- автоматически задает имя тега `img`.

Пример:

```php
$img = new Img();
```

---

## 8. Класс `A`

`A` — отдельный класс для ссылки.

Требования:

- наследуется от `PairTag`;
- автоматически задает имя тега `a`;
- атрибут `href` является обязательным.

Если у ссылки нет атрибута `href`, должна появиться ошибка:

```txt
Attribute href is required for A
```

Пример:

```php
$a = new A();

$a
    ->attr('href', 'http://yandex.ru')
    ->appendChild(new TextNode('Поиск'));
```

---

## 9. Проверка запрещенных атрибутов

Нужно реализовать проверку атрибутов.

Например, атрибут `onclick` должен быть запрещен.

Пример ошибки:

```txt
Attribute onclick is not allowed for SingleTag
```

Это нужно для защиты от вставки JavaScript в HTML.

Пример запрещенного атрибута:

```php
$hr = new SingleTag('hr');
$hr->attr('onclick', 'alert(1)');
```

Такой атрибут не должен считаться допустимым.

---

## 10. Цепочки вызовов

Методы `attr()` и `appendChild()` должны возвращать `$this`.

Это нужно, чтобы можно было писать так:

```php
$root
    ->attr('class', 'h1')
    ->appendChild($img)
    ->appendChild($hr)
    ->appendChild($h2);
```

---

## Пример тестового кода

```php
$img = new Img();
$hr = new SingleTag('hr');

$root = new PairTag('div');
$h2 = new PairTag('h2');

$t1 = new TextNode('Hello, <script>alert("haha");</script>, World!');
$t2 = new TextNode('Поиск');

$h2
    ->appendChild($t1)
    ->appendChild($t2);

$a = new A();

$a
    ->attr('href', 'http://yandex.ru')
    ->appendChild($t2);

$root
    ->attr('class', 'h1')
    ->appendChild($img)
    ->appendChild($hr)
    ->appendChild($h2)
    ->appendChild($a);

if ($root->isValid()) {
    echo $root->render();
} else {
    report();
}
```

## Что нужно проверить после выполнения

После выполнения задания нужно проверить следующие ситуации:

1. Корректный вывод HTML-разметки.
2. Одиночные теги выводятся без закрывающего тега.
3. Парные теги выводятся с открывающим и закрывающим тегом.
4. Вложенные элементы корректно отображаются.
5. Текст внутри `TextNode` экранируется.
6. JavaScript из текста не выполняется.
7. Пустой `TextNode` вызывает ошибку.
8. Ссылка `A` без `href` вызывает ошибку.
9. Запрещенный атрибут `onclick` вызывает ошибку.
10. Методы поддерживают цепочки вызовов.

## Пример полного сценария проверки

```php
$root = new PairTag('div');

$root
    ->attr('class', 'wrapper')
    ->appendChild(new Img())
    ->appendChild(new SingleTag('hr'))
    ->appendChild(
        (new PairTag('h2'))
            ->appendChild(new TextNode('Hello, <script>alert("haha");</script>, World!'))
    )
    ->appendChild(
        (new A())
            ->attr('href', 'http://yandex.ru')
            ->appendChild(new TextNode('Поиск'))
    );

if ($root->isValid()) {
    echo $root->render();
} else {
    report();
}
```

Ожидаемый смысл результата:

- `div` содержит вложенные элементы;
- `img` и `hr` выводятся как одиночные теги;
- `h2` выводится как парный тег;
- `a` выводится как ссылка;
- текст со скриптом не выполняется;
- ошибки выводятся через `report()`.
